Flujo Telegram para Diagnostico Agricola

Mapa interactivo de como quedaria el sistema: el agricultor envia una foto por Telegram, el orquestador la procesa con la CNN, agrega clima y responde con informacion base.

Mapa del proceso
1

Usuario envia foto

El agricultor toma una imagen de la hoja y la manda al bot de Telegram.

→
2

Bot recibe imagen

Telegram entrega el mensaje, el chat ID y el archivo de imagen.

→
3

n8n descarga archivo

El workflow toma el file_id de Telegram y obtiene el binario real de la foto.

→
4

n8n envia a FastAPI

La imagen se manda por HTTP POST al endpoint local del modelo.

5

CNN diagnostica

PyTorch procesa la imagen y devuelve clase probable y confianza.

→
6

n8n evalua confianza

Se define si el resultado es alto, medio o requiere inspeccion humana.

→
7

Consulta clima

OpenWeatherMap entrega temperatura, condicion y posible lluvia.

→
8

Gemini redacta

El agente combina diagnostico, confianza, clima y reglas agronomicas.

→
9

Respuesta por Telegram

El usuario recibe un mensaje breve con diagnostico, confianza, clima, recomendacion y advertencia.

Ruta feliz Foto clara, confianza alta y clima apto para recomendacion.
Ruta con alerta Confianza menor a 0.60: pedir nueva foto o revision humana.
Ruta climatica Si hay lluvia: no recomendar aplicacion inmediata de pesticidas.
Entrada

Usuario envia foto

El agricultor toma una foto de una hoja afectada y la envia al bot de Telegram. Idealmente debe ser una imagen nitida, con buena luz y una sola hoja como foco principal.

  • Dato principal: imagen JPG o PNG.
  • Dato secundario: chat ID para saber a quien responder.
  • Posible dato extra: ciudad o ubicacion si se quiere consultar clima local.

Ejemplo de mensaje final

Diagnostico: Oidio en Vid Confianza: 92.3% - Alta Clima actual: 18 C, nublado Recomendacion base: La imagen sugiere presencia de oidio. Revisar hojas cercanas y evitar aplicar tratamiento si hay lluvia o alta humedad. Nota: este resultado apoya la decision, pero no reemplaza una inspeccion profesional.
